## 环境概述

### 网络拓扑

```
Windows 物理机 (宿主机)
├── Hyper-V 虚拟交换机
│   ├── Nexus_WAN (外部交换机) ────────┐
│   └── Nexus_LAN_Isolated (内部交换机) ─┐
│                                        │
├── Ubuntu 虚拟机 (网关)                 │
│   ├── 网卡1 (eth0) ←──────────────────┘
│   └── 网卡2 (eth1) ←──────────────────┐
│                                        │
├── Windows 工作虚拟机 ←─────────────────┘
│   └── 唯一网卡
│
└── 物理机网卡 (可选)
    └── vEthernet (Nexus_LAN_Isolated)
        └── 192.168.100.200 (用于远程连接虚拟机)
```

### 系统要求

- **宿主机**：Windows 10/11 Pro（支持 Hyper-V）
- **Ubuntu 网关**：Ubuntu Server 22.04 LTS，至少 2GB 内存
- **工作虚拟机**：Windows 10/11，至少 2GB 内存

## 第一部分：Windows 宿主机准备

### 1.1 启用 Hyper-V

**方法一：通过控制面板**

1. 打开"控制面板" → "程序" → "启用或关闭 Windows 功能"
2. 勾选"Hyper-V"（包括所有子项）
3. 点击"确定"，重启电脑

**方法二：通过 PowerShell（管理员）**

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
```

### 1.2 创建虚拟交换机

#### 步骤 1：打开 Hyper-V 管理器

- 按 `Win + S` 搜索"Hyper-V 管理器"
- 或者在开始菜单找到"Windows 管理工具" → "Hyper-V 管理器"

#### 步骤 2：创建外部交换机（WAN）

1. 在右侧点击"虚拟交换机管理器"
2. 选择"新建虚拟网络交换机"
3. 类型选择"外部"
4. 点击"创建虚拟交换机"

**配置参数**：
```
名称：Nexus_WAN
连接类型：外部网络
选择网络适配器：[选择你的物理网卡，通常是连接互联网的那个]
✅ 勾选"允许管理操作系统共享此网络适配器"
```

5. 点击"确定"

⚠️ **注意**：创建外部交换机时，宿主机网络会短暂中断（约5秒）。

#### 步骤 3：创建内部交换机（LAN）

1. 再次点击"虚拟交换机管理器"
2. 选择"新建虚拟网络交换机"
3. 类型选择"内部"
4. 点击"创建虚拟交换机"

**配置参数**：
```
名称：Nexus_LAN_Isolated
连接类型：内部网络
```

5. 点击"确定"

### 1.3 配置宿主机内网网卡（可选）

⚠️ **重要说明**：这一步是**可选的**，仅在你需要从物理机远程连接（RDP）到内网虚拟机时才需要配置。

**为什么需要这一步？**
- 如果你想从物理机通过 `mstsc` (远程桌面) 连接到内网的 Windows 虚拟机
- 物理机需要和虚拟机在同一个网段（192.168.100.0/24）

**如果不需要远程连接，可以跳过此步骤。**

#### 配置步骤

1. 打开"网络和共享中心" → "更改适配器设置"
2. 找到 `vEthernet (Nexus_LAN_Isolated)` 网卡
3. 右键 → "属性"
4. 双击"Internet 协议版本 4 (TCP/IPv4)"

**配置参数**：
```
IP 地址：192.168.100.200
子网掩码：255.255.255.0
默认网关：留空
首选 DNS：留空
```

5. 点击"确定"

**验证**：
```cmd
ping 192.168.100.200
# 应该能 ping 通自己
```

## 第二部分：Ubuntu 网关虚拟机配置

### 2.1 创建 Ubuntu 虚拟机

#### 步骤 1：新建虚拟机

1. 在 Hyper-V 管理器中，右键点击你的主机名
2. 选择"新建" → "虚拟机"
3. 点击"下一步"

#### 步骤 2：配置虚拟机

**指定名称和位置**：
```
名称：NexusRoute-Gateway
位置：[选择存储位置]
```

**指定代数**：
```
选择：第 2 代
```

**分配内存**：
```
启动内存：2048 MB (2GB)
✅ 勾选"为此虚拟机使用动态内存"
```

**配置网络**：
```
连接：Nexus_WAN
```
⚠️ 注意：这里先只配置一个网卡，第二个网卡稍后添加。

**连接虚拟硬盘**：
```
创建虚拟硬盘
名称：NexusRoute-Gateway.vhdx
位置：[默认]
大小：20 GB
```

**安装选项**：
```
从可启动映像文件安装操作系统
映像文件(.iso)：[选择 Ubuntu Server 22.04 ISO]
```

#### 步骤 3：添加第二个网卡

1. 虚拟机创建完成后，右键虚拟机 → "设置"
2. 点击"添加硬件" → "网络适配器" → "添加"
3. 虚拟交换机选择：`Nexus_LAN_Isolated`
4. 点击"确定"

**最终网卡配置**：
```
网络适配器 1：Nexus_WAN (外网)
网络适配器 2：Nexus_LAN_Isolated (内网)
```

### 2.2 安装 Ubuntu Server 22.04

#### 步骤 1：启动虚拟机

1. 右键虚拟机 → "连接"
2. 点击"启动"
3. 进入 Ubuntu 安装界面

#### 步骤 2：安装过程

**语言选择**：
```
English (推荐，避免中文乱码)
```

**键盘布局**：
```
English (US)
```

**网络配置**（重要！）：

你会看到两个网卡：
- `enp1s0` 或 `eth0` (对应 Nexus_WAN)
- `enp2s0` 或 `eth1` (对应 Nexus_LAN_Isolated)

**配置 eth0 (外网)**：
```
选择：DHCP (自动获取)
或者配置静态 IP（如果你的网络需要）
```

**配置 eth1 (内网)**：

⚠️ **避坑关键**：eth1 因为没有 DHCP 服务器会卡住，需要手动配置！

1. 选中 `enp2s0` 或 `eth1`
2. 按 `Enter` 编辑
3. 选择 `Edit IPv4`
4. 选择 `Manual`

**配置参数**：
```
Subnet: 192.168.100.0/24
Address: 192.168.100.1
Gateway: 留空（不填）
Name servers: 留空（不填）
Search domains: 留空（不填）
```

5. 选择 `Save`

**其他安装选项**：
```
代理：留空
镜像源：默认（或选择国内镜像）
存储：使用整个磁盘
用户名：[自定义]
密码：[自定义]
SSH：✅ 勾选 "Install OpenSSH server"
```

#### 步骤 3：完成安装

1. 等待安装完成（约 10-15 分钟）
2. 选择 "Reboot Now"
3. 重启后登录系统

### 2.3 验证网络配置

登录 Ubuntu 后，验证网卡配置：

```bash
# 查看网卡
ip addr show

# 应该看到：
# eth0: 有 IP 地址（DHCP 分配或静态配置）
# eth1: 192.168.100.1/24

# 测试外网连接
ping -c 3 8.8.8.8

# 测试 DNS
ping -c 3 google.com
```

### 2.4 配置 netplan（如果需要修改）

如果安装时配置有误，可以手动修改：

```bash
# 编辑 netplan 配置
sudo nano /etc/netplan/00-installer-config.yaml
```

**配置内容**：
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: true
    eth1:
      dhcp4: false
      addresses:
        - 192.168.100.1/24
```

**应用配置**：
```bash
sudo netplan apply
```

## 第三部分：Windows 工作虚拟机配置

### 3.1 创建 Windows 虚拟机

#### 步骤 1：新建虚拟机

1. 在 Hyper-V 管理器中，右键点击主机名
2. 选择"新建" → "虚拟机"

#### 步骤 2：配置虚拟机

**指定名称和位置**：
```
名称：NexusRoute-Client-01
位置：[选择存储位置]
```

**指定代数**：
```
选择：第 2 代
```

**分配内存**：
```
启动内存：4096 MB (4GB)
✅ 勾选"为此虚拟机使用动态内存"
```

**配置网络**：
```
连接：Nexus_LAN_Isolated
```

⚠️ **重要**：只连接内部交换机，不要连接外部交换机！

**连接虚拟硬盘**：
```
创建虚拟硬盘
名称：NexusRoute-Client-01.vhdx
大小：60 GB
```

**安装选项**：
```
从可启动映像文件安装操作系统
映像文件(.iso)：[选择 Windows 10/11 ISO]
```

### 3.2 安装 Windows

1. 启动虚拟机
2. 按照正常流程安装 Windows
3. 安装完成后，检查网络配置

### 3.3 验证网络配置

#### 步骤 1：检查网络状态

安装完成后，Windows 会显示"未识别的网络"或"无 Internet 访问"，这是**正常的**！

因为：
1. Ubuntu 网关还没有安装 NexusRoute
2. dnsmasq 还没有启动，无法分配 IP

#### 步骤 2：手动配置临时 IP（可选）

如果你想在安装 NexusRoute 前测试连通性：

1. 打开"网络和共享中心" → "更改适配器设置"
2. 右键"以太网" → "属性"
3. 双击"Internet 协议版本 4 (TCP/IPv4)"

**配置参数**：
```
IP 地址：192.168.100.50
子网掩码：255.255.255.0
默认网关：192.168.100.1
首选 DNS：192.168.100.1
```

4. 点击"确定"

**测试连通性**：
```cmd
ping 192.168.100.1
# 应该能 ping 通网关
```

⚠️ **注意**：安装 NexusRoute 后，需要改回"自动获取 IP 地址"。

## 第四部分：验证与测试

### 4.1 网络连通性测试

#### 在 Ubuntu 网关上测试

```bash
# 测试外网
ping -c 3 8.8.8.8

# 测试内网接口
ip addr show eth1
# 应该显示 192.168.100.1/24
```

#### 在 Windows 虚拟机上测试

```cmd
# 测试网关连通性
ping 192.168.100.1

# 查看网络配置
ipconfig /all
```

### 4.2 安装 NexusRoute

在 Ubuntu 网关上执行：

```bash
# 下载项目
git clone https://github.com/Kxiandaoyan/NexusRoute.git
cd NexusRoute

# 运行安装脚本
chmod +x install.sh
sudo ./install.sh
```

安装完成后，Windows 虚拟机应该能够：
1. 自动获取 IP（192.168.100.50-99）
2. 在管理后台看到待审批设备
3. 批准后获得永久 IP（192.168.100.10-209）

### 4.3 防漏油测试

```bash
# 在 Ubuntu 网关上停止代理
sudo systemctl stop xray-user1

# 在 Windows 虚拟机上测试
ping 8.8.8.8
# 应该超时（无法连接）

# 恢复代理
sudo systemctl start xray-user1
```

## 常见问题

### Q1: Ubuntu 安装时 eth1 一直卡住怎么办？

A: 这是因为 eth1 没有 DHCP 服务器。按照 [2.2 安装 Ubuntu](#步骤-2安装过程) 中的步骤手动配置为静态 IP。

### Q2: Windows 虚拟机无法获取 IP？

A: 检查：
1. 虚拟机网卡是否连接到 `Nexus_LAN_Isolated`
2. Ubuntu 网关的 dnsmasq 服务是否运行：`sudo systemctl status dnsmasq`
3. Ubuntu 的 eth1 是否配置为 192.168.100.1

### Q3: 物理机无法远程连接到虚拟机？

A: 检查：
1. 物理机的 `vEthernet (Nexus_LAN_Isolated)` 是否配置为 192.168.100.200
2. 虚拟机是否已获取 IP
3. 虚拟机防火墙是否允许远程桌面

### Q4: 外部交换机创建后宿主机断网？

A: 这是正常的，创建外部交换机时会短暂中断网络（5-10秒）。如果长时间断网，检查是否选择了正确的物理网卡。
