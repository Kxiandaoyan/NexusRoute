## 环境概述

### 网络拓扑

```
Windows/Linux 物理机 (宿主机)
├── VMware Workstation/Fusion
│   ├── VMnet0 (桥接模式) ────────────────┐
│   └── VMnet2 (仅主机模式) ───────────────┐
│                                          │
├── Ubuntu 虚拟机 (网关)                   │
│   ├── 网卡1 (ens33) ←────────────────────┘
│   └── 网卡2 (ens34) ←────────────────────┐
│                                          │
└── Windows 工作虚拟机 ←───────────────────┘
    └── 唯一网卡
```

### 系统要求

- **宿主机**：Windows/Linux/macOS（支持 VMware Workstation/Fusion/Player）
- **Ubuntu 网关**：Ubuntu Server 22.04 LTS，至少 2GB 内存
- **工作虚拟机**：Windows 10/11，至少 2GB 内存

### 适用版本

- VMware Workstation Pro/Player (Windows/Linux)
- VMware Fusion (macOS)
- VMware ESXi (企业环境)

## 第一部分：VMware 虚拟网络准备

### 1.1 打开虚拟网络编辑器

#### Windows/Linux

1. 打开 VMware Workstation
2. 点击菜单栏"编辑" → "虚拟网络编辑器"
3. 如果提示需要管理员权限，点击"更改设置"

#### macOS

1. 打开 VMware Fusion
2. 点击菜单栏"VMware Fusion" → "偏好设置"
3. 选择"网络"选项卡

### 1.2 配置桥接网络（WAN）

#### 步骤 1：检查 VMnet0

VMnet0 是默认的桥接网络，通常已经存在。

**配置参数**：
```
名称：VMnet0
类型：桥接模式
桥接到：自动（或选择你的物理网卡）
```

⚠️ **注意**：如果你的电脑有多个网卡（有线+无线），建议手动选择连接互联网的那个。

#### 步骤 2：验证桥接网络

1. 在虚拟网络编辑器中选择 VMnet0
2. 确认"桥接模式"已勾选
3. 点击"确定"

### 1.3 配置仅主机网络（LAN）

#### 步骤 1：选择或创建 VMnet

**推荐使用 VMnet2**（避免与默认的 VMnet1 和 VMnet8 冲突）

1. 在虚拟网络编辑器中选择 VMnet2
2. 如果 VMnet2 不存在，点击"添加网络"

#### 步骤 2：配置 VMnet2

**配置参数**：
```
名称：VMnet2
类型：仅主机模式
子网 IP：192.168.100.0
子网掩码：255.255.255.0
```

⚠️ **重要**：取消勾选"使用本地 DHCP 服务将 IP 地址分配给虚拟机"

**为什么要禁用 DHCP？**
- NexusRoute 使用自己的 dnsmasq 作为 DHCP 服务器
- VMware 的 DHCP 会冲突

#### 步骤 3：配置宿主机虚拟网卡（可选）

如果你想从物理机远程连接（RDP）到内网虚拟机：

**Windows/Linux**：
1. 在虚拟网络编辑器中选择 VMnet2
2. 勾选"将主机虚拟适配器连接到此网络"
3. 点击"确定"

**配置宿主机 IP**：
1. 打开网络设置
2. 找到 `VMware Network Adapter VMnet2`
3. 配置静态 IP：
   ```
   IP 地址：192.168.100.200
   子网掩码：255.255.255.0
   默认网关：留空
   DNS：留空
   ```

**macOS**：
1. 在 Fusion 网络设置中选择 VMnet2
2. 勾选"允许此网络上的虚拟机连接到外部网络"（取消勾选）
3. 勾选"将我的 Mac 连接到此网络"

**验证**：
```bash
# Windows
ping 192.168.100.200

# macOS/Linux
ping 192.168.100.200
```

## 第二部分：Ubuntu 网关虚拟机配置

### 2.1 创建 Ubuntu 虚拟机

#### 步骤 1：新建虚拟机

1. 打开 VMware
2. 点击"创建新的虚拟机"
3. 选择"典型（推荐）"或"自定义（高级）"

#### 步骤 2：选择安装源

**安装程序光盘映像文件 (iso)**：
```
选择 Ubuntu Server 22.04 ISO 文件
```

⚠️ **注意**：如果 VMware 提示"检测到 Easy Install"，建议选择"稍后安装操作系统"以获得更多控制。

#### 步骤 3：配置虚拟机

**虚拟机名称**：
```
名称：NexusRoute-Gateway
位置：[选择存储位置]
```

**处理器配置**：
```
处理器数量：1
每个处理器的核心数量：2
```

**内存**：
```
内存：2048 MB (2GB)
```

**网络类型**：
```
网络连接：桥接网络（VMnet0）
```

⚠️ 注意：这里先只配置一个网卡，第二个网卡稍后添加。

**磁盘**：
```
磁盘类型：SCSI（推荐）
磁盘大小：20 GB
存储为单个文件（推荐）
```

#### 步骤 4：添加第二个网卡

1. 虚拟机创建完成后，右键虚拟机 → "设置"
2. 点击"添加" → "网络适配器" → "下一步"
3. 网络连接选择：`自定义：特定虚拟网络`
4. 选择 `VMnet2 (仅主机模式)`
5. 点击"完成"

**最终网卡配置**：
```
网络适配器 1：桥接模式 (VMnet0) - 外网
网络适配器 2：仅主机模式 (VMnet2) - 内网
```

### 2.2 安装 Ubuntu Server 22.04

#### 步骤 1：启动虚拟机

1. 选中虚拟机
2. 点击"开启此虚拟机"
3. 进入 Ubuntu 安装界面

#### 步骤 2：安装过程

**语言选择**：
```
English (推荐，避免中文乱码)
```

**键盘布局**：
```
English (US)
```

**网络配置**（重要！）：

你会看到两个网卡：
- `ens33` 或 `enp0s3` (对应 VMnet0 桥接)
- `ens34` 或 `enp0s8` (对应 VMnet2 仅主机)

**配置 ens33 (外网)**：
```
选择：DHCP (自动获取)
或者配置静态 IP（如果你的网络需要）
```

**配置 ens34 (内网)**：

⚠️ **避坑关键**：ens34 因为没有 DHCP 服务器会卡住，需要手动配置！

1. 选中 `ens34` 或 `enp0s8`
2. 按 `Enter` 编辑
3. 选择 `Edit IPv4`
4. 选择 `Manual`

**配置参数**：
```
Subnet: 192.168.100.0/24
Address: 192.168.100.1
Gateway: 留空（不填）
Name servers: 留空（不填）
Search domains: 留空（不填）
```

5. 选择 `Save`

**其他安装选项**：
```
代理：留空
镜像源：默认（或选择国内镜像）
存储：使用整个磁盘
用户名：[自定义]
密码：[自定义]
SSH：✅ 勾选 "Install OpenSSH server"
```

#### 步骤 3：完成安装

1. 等待安装完成（约 10-15 分钟）
2. 选择 "Reboot Now"
3. 重启后登录系统

### 2.3 验证网络配置

登录 Ubuntu 后，验证网卡配置：

```bash
# 查看网卡
ip addr show

# 应该看到：
# ens33: 有 IP 地址（DHCP 分配或静态配置）
# ens34: 192.168.100.1/24

# 测试外网连接
ping -c 3 8.8.8.8

# 测试 DNS
ping -c 3 google.com
```

### 2.4 配置 netplan（如果需要修改）

如果安装时配置有误，可以手动修改：

```bash
# 编辑 netplan 配置
sudo nano /etc/netplan/00-installer-config.yaml
```

**配置内容**：
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    ens33:
      dhcp4: true
    ens34:
      dhcp4: false
      addresses:
        - 192.168.100.1/24
```

**应用配置**：
```bash
sudo netplan apply
```

## 第三部分：Windows 工作虚拟机配置

### 3.1 创建 Windows 虚拟机

#### 步骤 1：新建虚拟机

1. 在 VMware 中点击"创建新的虚拟机"
2. 选择"典型（推荐）"

#### 步骤 2：配置虚拟机

**安装源**：
```
安装程序光盘映像文件 (iso)：[选择 Windows 10/11 ISO]
```

**虚拟机名称**：
```
名称：NexusRoute-Client-01
位置：[选择存储位置]
```

**磁盘大小**：
```
最大磁盘大小：60 GB
存储为单个文件（推荐）
```

**内存**：
```
内存：4096 MB (4GB)
```

**网络连接**：
```
网络适配器：自定义：特定虚拟网络
选择：VMnet2 (仅主机模式)
```

⚠️ **重要**：只连接 VMnet2，不要连接桥接网络！

#### 步骤 3：自定义硬件（可选）

点击"自定义硬件"可以调整：
- 处理器数量（建议 2 核）
- 内存大小（建议 4GB）
- 移除不需要的设备（打印机、声卡等）

### 3.2 安装 Windows

1. 启动虚拟机
2. 按照正常流程安装 Windows
3. 安装完成后，检查网络配置

### 3.3 安装 VMware Tools（推荐）

**为什么需要 VMware Tools？**
- 更好的图形性能
- 鼠标无缝切换
- 文件拖放功能
- 时间同步

**安装步骤**：
1. 在 VMware 菜单中选择"虚拟机" → "安装 VMware Tools"
2. 在 Windows 中打开光驱
3. 运行 `setup.exe`
4. 按提示完成安装
5. 重启虚拟机

### 3.4 验证网络配置

#### 步骤 1：检查网络状态

安装完成后，Windows 会显示"未识别的网络"或"无 Internet 访问"，这是**正常的**！

因为：
1. Ubuntu 网关还没有安装 NexusRoute
2. dnsmasq 还没有启动，无法分配 IP

#### 步骤 2：手动配置临时 IP（可选）

如果你想在安装 NexusRoute 前测试连通性：

1. 打开"网络和共享中心" → "更改适配器设置"
2. 右键"以太网" → "属性"
3. 双击"Internet 协议版本 4 (TCP/IPv4)"

**配置参数**：
```
IP 地址：192.168.100.50
子网掩码：255.255.255.0
默认网关：192.168.100.1
首选 DNS：192.168.100.1
```

4. 点击"确定"

**测试连通性**：
```cmd
ping 192.168.100.1
# 应该能 ping 通网关
```

⚠️ **注意**：安装 NexusRoute 后，需要改回"自动获取 IP 地址"。

## 第四部分：验证与测试

### 4.1 网络连通性测试

#### 在 Ubuntu 网关上测试

```bash
# 测试外网
ping -c 3 8.8.8.8

# 测试内网接口
ip addr show ens34
# 应该显示 192.168.100.1/24
```

#### 在 Windows 虚拟机上测试

```cmd
# 测试网关连通性
ping 192.168.100.1

# 查看网络配置
ipconfig /all
```

### 4.2 安装 NexusRoute

在 Ubuntu 网关上执行：

```bash
# 下载项目
git clone https://github.com/Kxiandaoyan/NexusRoute.git
cd NexusRoute

# 运行安装脚本
chmod +x install.sh
sudo ./install.sh
```

安装完成后，Windows 虚拟机应该能够：
1. 自动获取 IP（192.168.100.50-99）
2. 在管理后台看到待审批设备
3. 批准后获得永久 IP（192.168.100.10-209）

### 4.3 防漏油测试

```bash
# 在 Ubuntu 网关上停止代理
sudo systemctl stop xray-user1

# 在 Windows 虚拟机上测试
ping 8.8.8.8
# 应该超时（无法连接）

# 恢复代理
sudo systemctl start xray-user1
```

## 常见问题

### Q1: Ubuntu 安装时 ens34 一直卡住怎么办？

A: 这是因为 ens34 没有 DHCP 服务器。按照 [2.2 安装 Ubuntu](#步骤-2安装过程) 中的步骤手动配置为静态 IP。

### Q2: Windows 虚拟机无法获取 IP？

A: 检查：
1. 虚拟机网卡是否连接到 VMnet2
2. Ubuntu 网关的 dnsmasq 服务是否运行：`sudo systemctl status dnsmasq`
3. Ubuntu 的 ens34 是否配置为 192.168.100.1

### Q3: 物理机无法远程连接到虚拟机？

A: 检查：
1. 物理机的 `VMware Network Adapter VMnet2` 是否配置为 192.168.100.200
2. 虚拟机是否已获取 IP
3. 虚拟机防火墙是否允许远程桌面

### Q4: VMnet2 的 DHCP 无法禁用？

A: 在虚拟网络编辑器中：
1. 选择 VMnet2
2. 取消勾选"使用本地 DHCP 服务..."
3. 点击"应用" → "确定"
4. 重启 VMware 服务（Windows）或重启电脑（macOS）

### Q5: ESXi 环境如何配置？

A: ESXi 环境配置类似，但需要：
1. 创建两个端口组（一个连接物理网络，一个隔离网络）
2. 隔离网络的端口组不要连接上行链路
3. 其他步骤与 Workstation 相同

## VMware 特定优化

### 性能优化

**虚拟机设置**：
```
处理器：启用虚拟化 Intel VT-x/EPT 或 AMD-V/RVI
内存：预留所有客户机内存（可选）
磁盘：使用 SCSI 控制器（性能更好）
```

**网络优化**：
```
网络适配器类型：VMXNET3（性能最佳）
```

### 快照管理

建议在以下时间点创建快照：
1. Ubuntu 安装完成后
2. NexusRoute 安装完成后
3. 添加第一个代理节点后

**创建快照**：
1. 右键虚拟机 → "快照" → "拍摄快照"
2. 输入快照名称和描述
3. 点击"拍摄快照"
